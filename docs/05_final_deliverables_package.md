# 大创结题交付包（执行版）

## 1. 明确结论

本项目交付分为“系统演示、技术文档、学术产出、竞赛材料”四大包，保证可验收、可复现、可展示。

## 2. 交付清单

### 2.1 系统演示包

- Web演示端（病例上传、风险评分、热图可视化、解释文本）
- 推理后端服务（FastAPI）
- 模型文件：
  - 全模态教师模型（研究版）
  - 轻量学生模型（部署版，ONNX）
- 演示视频（3-5分钟）

### 2.2 技术文档包

- 项目背景与需求说明
- 数据治理与标注规范
- 模型设计与训练配置
- 评测报告（含消融实验与分层分析）
- 部署手册（环境、启动、接口、排错）

### 2.3 学术与知识产权包

- 大创中期/结题报告（学校模板）
- 会议论文或技术报告初稿（可选）
- 软著准备材料（可选）

### 2.4 竞赛转化包

- 项目路演PPT（10-15页）
- 3分钟电梯演讲稿
- 评委问答清单（技术、医学、伦理、落地）

## 3. 验收指标建议

- 功能验收：三模态输入联调成功，输出风险+热图+解释文本
- 算法验收：AUC、敏感度、推理延迟达到目标阈值
- 工程验收：一键部署脚本可复现核心流程
- 文档验收：关键文档完整且可追溯版本

## 4. 时间安排（建议）

- 第10月：功能冻结与性能冲刺
- 第11月：文档整理与路演材料打磨
- 第12月：结题答辩、竞赛提交、成果归档

## 5. 风险与后续动作

- 风险1：系统可用但展示效果一般  
  后续动作：提前准备高质量Demo病例与解释可视化模板。

- 风险2：文档分散导致答辩准备低效  
  后续动作：统一归档到`docs/`并设置版本标记。
