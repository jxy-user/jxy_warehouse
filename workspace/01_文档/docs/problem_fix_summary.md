- 修复方案：

  - 新增多模态基线与离线评测文档
  - 新增小样本与轻量化部署文档
  - 新增结题交付包文档
  - 新增云开发数据库文档与云函数权限文档
- 影响范围：`workspace/01_文档/docs/`目录下项目设计与治理文档体系。
- 回归验证：检查文档完整性、结构一致性、可执行性与风险条目齐备。
- 预防动作：后续开发新增云函数前，先更新数据库/权限文档；每次修复同步更新本记录。

### [2026-04-16] 仅支持Mock数据，缺少真实数据与图片上传推理

- 问题现象：初版仅支持mock输入，无法直接接入CSV真实数据，也不能上传影像文件进行推理。
- 根因分析：MVP阶段优先打通训练链路，未补齐真实数据读取与接口形态。
- 修复方案：
  - 新增CSV多模态数据集读取（`CSVMultiModalDataset`）
  - 配置新增`data.mode/csv_path/image_root`，支持mock与csv切换
  - 推理接口新增`/infer_with_image`，支持上传图片并返回热图路径
  - README新增中文化真实数据接入说明
  - 同步更新云数据库与云函数权限文档（新增可解释产物集合与函数权限）
- 影响范围：`src/data/dataset.py`、`src/config/default.yaml`、`src/api/infer_api.py`、`README.md`、云开发文档。
- 回归验证：代码结构完整，接口输入输出字段齐全；当前环境因缺依赖未执行完整运行验证。
- 预防动作：后续每新增推理能力，必须同步更新数据库文档、权限文档与问题总结文档。

### [2026-04-16] 图片推理仅为占位热图，缺少真实可解释算法

- 问题现象：`/infer_with_image`返回的是占位热图，未体现真实可解释机制。
- 根因分析：为先打通上传链路，暂用灰度图复用占位，未接入Grad-CAM反向传播流程。
- 修复方案：
  - 在推理接口中接入Grad-CAM计算流程（前向激活+反向梯度）
  - 按最高风险类别自动生成Grad-CAM叠加图并落盘
  - 返回说明改为“已生成Grad-CAM热图”
  - 同步更新数据库文档与云函数权限文档中的可解释产物定义
- 影响范围：`src/api/infer_api.py`、`src/models/mmca_net.py`、`README.md`、云开发文档。
- 回归验证：`python -m compileall src`语法通过；接口字段与落盘路径完整。
- 预防动作：后续新增“可解释性”相关能力时，禁止仅占位实现，需提供最小可验证算法链路。

### [2026-04-16] 页面接口地址写死，无法切换本地与云端

- 问题现象：前端请求固定写死`http://127.0.0.1:8000`，部署到云端后需要手改代码。
- 根因分析：交互页迭代时优先联调本地接口，未抽离可配置地址。
- 修复方案：
  - 新增“接口基地址”输入框与“保存接口地址”按钮
  - 健康检查与上传推理统一使用动态基地址
  - 使用localStorage持久化接口地址，刷新后自动恢复
- 影响范围：`workspace/01_文档/frontend_dark.html`、`workspace/01_文档/README.md`。
- 回归验证：页面加载后可读取已保存地址；健康检查与推理请求可随地址切换。
- 预防动作：后续新增前端接口调用时，默认采用可配置基础URL，避免硬编码。

### [2026-04-16] 频繁切换地址效率低，缺少快捷历史

- 问题现象：虽支持手动填写接口地址，但本地/云端/测试地址切换仍需反复输入。
- 根因分析：仅有单值保存逻辑，没有地址历史管理与快速选择入口。
- 修复方案：
  - 新增“最近使用地址（最多3个）”下拉框
  - 新增历史读写逻辑（localStorage 持久化）
  - 保存地址、健康检查成功、推理成功时自动更新历史
  - 选择历史项后自动填充当前接口基地址
- 影响范围：`workspace/01_文档/frontend_dark.html`、`workspace/01_文档/README.md`。
- 回归验证：历史列表最多保留3项，且可快速切换并生效。
- 预防动作：后续涉及环境切换的前端配置，优先提供历史与快速选择能力。
