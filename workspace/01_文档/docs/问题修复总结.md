### [2026-04-22] 增加 `route_infer` 自动路由，前端按问题自动分流模块

- 问题现象：用户在前端提问时，无法根据问题类型自动切换骨骼影像与坐姿模块，需手动改`active_module`。
- 根因分析：后端仅提供固定模块推理接口（`/infer*`），缺少文本意图路由层；前端也没有显示路由结果。
- 修复方案：
  - 在 `infer_api.py` 新增 `POST /route_infer`：基于关键词与是否上传影像进行路由，输出 `route_module`、`route_reason`、`route_confidence`；
  - 路由到骨骼时复用 MMC 推理流程，路由到坐姿时返回占位结构（后续可替换真实算法）；
  - 前端 `frontend_dark.html` 改为调用 `/route_infer`，并展示自动路由结果；
  - 同步更新 README、云函数权限文档、云数据库文档。
- 影响范围：`workspace/02_工程/src/api/infer_api.py`、`workspace/01_文档/frontend_dark.html`、`workspace/01_文档/README.md`、云开发文档。
- 回归验证：上传或仅文本提问均可获得自动路由响应；返回包含模块、路由原因、置信度与结果体。
- 预防动作：后续新增模块先接入 `route_infer` 路由器再开放前端入口，保证用户零配置使用。

### [2026-04-22] 新增 posture_stub 坐姿占位模块与 pipeline 分流

- 问题现象：申报书含坐姿检测，但不宜与骨骼 MMC 混在同一训练脚本；需独立扩展点。
- 根因分析：单一路径难以同时承载影像多标签与视频/姿态管线。
- 修复方案：
  - 新增 `modules/posture_stub.yaml`（`pipeline: posture_stub`）及示例 `train_posture_stub.csv`；
  - `bone`/`chest_mvp` 增加 `pipeline: mmc_net`；`infer_api` 按 pipeline 决定是否实例化 MMCANet；
  - 常驻路由 `GET /posture/info`、`POST /posture/analyze`（配置通过 `load_module_overlay` 固定读 posture_stub）；
  - `train`/`evaluate` 在 `posture_stub` 下退出并提示切换模块。
- 影响范围：`utils.py`、`infer_api.py`、`train.py`、`evaluate.py`、`modules/*`、`docs/10_*`。
- 回归验证：`active_module: bone` 时仍可调用 `/posture/*`；切换为 `posture_stub` 时主推理为占位。
- 预防动作：姿态算法落地时新建 pipeline 或替换 stub 模块文件，避免污染 `mmc_net` 代码路径。

### [2026-04-22] 引入 active_module 任务模块化，保留胸片能力并默认骨骼任务

- 问题现象：此前将默认类别改为骨骼后，原版胸片五类联调能力与单一 `train.csv` 路径不便共存；后续再加功能容易反复改主配置。
- 根因分析：任务差异（类别、CSV、权重路径）与应用层代码耦合在同一 `default.yaml`。
- 修复方案：
  - 新增 `load_resolved_config`，按 `active_module` 合并 `src/config/modules/<id>.yaml`；
  - 内置 `bone`、`chest_mvp` 两模块：独立 `class_names`、`csv_path`、`checkpoint_path`、`save_path`；
  - 拆分示例数据为 `train_bone.csv`、`train_chest_mvp.csv`，删除笼统的 `train.csv`；
  - 训练、评测、推理入口统一使用合并后配置；`GET /health` 返回模块与类别列表。
- 影响范围：`utils.py`、`default.yaml`、`infer_api.py`、`train.py`、`evaluate.py`、`config/modules/*`、`README`。
- 回归验证：切换 `active_module` 后类别与路径应随模块变化；mock/csv 训练与 API 启动无语法错误。
- 预防动作：新能力优先新增 `modules/新模块.yaml`，避免再改核心 Python。
