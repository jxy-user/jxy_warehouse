# 挑战杯科技发明制作类申报书（灵枢AI）

## 一、项目名称

灵枢AI——青少年骨骼健康智能评估系统（基于骨骼医疗影像与坐姿规范检测的融合干预平台）

## 二、项目类别与定位

- 申报类别：挑战杯科技发明制作类
- 项目定位：围绕青少年骨骼健康，构建“医院筛查 + 校园监测 + 家庭随访”一体化智能评估系统
- 本阶段边界：完成申报方案与技术路线论证；坐姿检测功能后续进入代码迭代阶段

## 三、立项背景与研究意义

### 3.1 背景痛点

1. **生活痛点**：青少年长期伏案学习与电子设备使用增加，低头、弯腰、久坐问题普遍，但学校和家庭缺少连续、客观、低干扰的体态监测工具。  
2. **医疗痛点**：骨折恢复、骨龄偏差、脊柱体态异常等问题在基层场景筛查压力大，优质影像判读资源分布不均。  
3. **管理痛点**：家长感知、学校管理与医疗评估相互割裂，难形成“早发现-早干预-再评估”的闭环。  
4. **信任痛点**：现有AI方案常给出结论但缺少可解释证据，家长与教师对结果可信度不足。  

### 3.2 项目意义  

- 医疗价值：提升骨骼影像辅助判读效率，提前识别高风险人群并辅助分层处置。  
- 教育价值：把“被动提醒”升级为“数据化干预”，为班级与家庭提供可执行改进建议。  
- 社会价值：降低家校医协同沟通成本，形成可复制、可推广的青少年骨骼健康管理模式。  

### 3.3 价值主张（竞赛叙事总论）

本项目不是“单个AI模型”，而是面向现实生活痛点的健康管理方案：通过骨骼影像识别生理风险，通过坐姿检测识别行为风险，再借助问答交互与可解释证据建立家校医协同信任，最终形成“筛查-干预-复评”的闭环服务。

## 四、国内外研究现状与问题分析

### 4.1 研究现状

- 医疗影像AI在分类、分割方面已形成大量研究，骨折识别和骨龄评估任务已有公开模型与数据基础。  
- 姿态估计领域（关键点检测）技术成熟，可支持实时人体姿态识别。  

### 4.2 现有不足

- 医疗影像系统与日常行为监测系统多为割裂建设，缺少健康闭环。  
- 部分模型可解释性不足，难以用于教学与临床沟通。  
- 轻量部署和可持续维护能力不足，不利于校园和基层推广。  

### 4.3 本项目切入点

构建“骨骼影像风险分层 + 坐姿行为干预”的双模块联动系统，实现“筛查-干预-复评”闭环。

## 五、项目目标与创新点

## 5.1 总体目标

在现有医学影像工程能力基础上，形成可演示、可验证、可迭代的青少年骨骼健康智能评估原型。

### 5.2 分目标

- 目标1：实现骨骼医疗影像辅助分析（骨折风险、骨龄偏差、脊柱体态风险提示）。  
- 目标2：实现青少年坐姿规范性检测与实时提醒策略。  
- 目标3：形成统一评估报告，支持家校医场景复盘与随访。  

### 5.3 创新点

1. **融合创新（从“单点识别”到“联合评估”）**  
   - 现状不足：影像系统与行为监测工具通常独立运行。  
   - 本项目做法：将骨骼影像风险与坐姿行为风险统一到一个联合评分框架。  
   - 可验证指标：高风险个体提前发现率、联合评分与单模块评分提升幅度。  
2. **可信创新（从“黑箱结论”到“证据可读”）**  
   - 现状不足：家长和学校对AI结论可解释性要求高。  
   - 本项目做法：输出热图证据、风险分层原因、问答式解释。  
   - 可验证指标：解释可理解度问卷、复核一致率、用户采纳率。  
3. **落地创新（从“实验室模型”到“校园可用”）**  
   - 现状不足：部分方案部署复杂、维护成本高。  
   - 本项目做法：模块化插件接入、轻量化部署、分阶段迭代。  
   - 可验证指标：单例推理时延、部署成本、运维工作量。  
4. **治理创新（从“功能先行”到“治理先行”）**  
   - 现状不足：竞赛项目常忽视权限、审计与问题复盘。  
   - 本项目做法：把数据库文档、权限文档、问题修复沉淀机制纳入研发流程。  
   - 可验证指标：文档同步率、审计覆盖率、问题复发率。  

## 六、技术路线与系统架构

### 6.1 整体技术路线

1. 数据治理与标签标准化  
2. 骨骼影像基线模型训练与离线评测  
3. 坐姿检测规则设计与风险评分策略  
4. 双模块结果融合与报告生成  
5. 小样本迭代与轻量化部署优化  

### 6.2 系统架构（申报级）

- 数据层：公开骨骼影像数据 + 后续合规校内数据（脱敏后）  
- 模型层：影像编码器、多模态融合头、姿态关键点估计模块  
- 应用层：风险评分、可解释展示、异常提醒、随访记录  
- 治理层：权限矩阵、审计日志、问题修复文档沉淀机制  

### 6.3 与既有功能融合（避免推翻重做）

- 复用既有问答交互能力：作为统一入口，降低学生与家长使用门槛。  
- 复用既有风险分数与可解释热图能力：用于骨骼影像结果展示与证据支撑。  
- 复用既有接口安全与治理能力：延续鉴权、限流、权限矩阵、审计日志策略。  
- 新增坐姿检测模块采用插件化接入：与现有后端推理链路并行，不破坏当前主流程。  
- 输出统一报告：整合“影像生理风险 + 行为姿态风险 + 问答建议”，形成闭环。  

## 七、核心功能设计

### 7.1 骨骼医疗影像模块（重点）

#### 任务定义

- 骨折辅助识别：识别疑似骨折区域并给出风险级别。  
- 骨龄评估：提供骨龄偏差区间，用于生长发育趋势提示。  
- 脊柱体态风险提示：结合影像信息给出侧弯风险分层建议。  

#### 技术方案

- 输入：骨骼影像 + 基础结构化特征（年龄、性别、既往史等）  
- 编码：影像主干网络提取视觉特征，结构化特征经轻量MLP编码  
- 融合：跨注意力/门控融合获得患者级风险表示  
- 输出：多任务风险分数 + 解释热图（Grad-CAM类） + 文字证据摘要  

#### 量化指标（申报口径）

- 分类指标：AUC、Sensitivity、Specificity、F1  
- 效率指标：单例推理延迟、吞吐能力  
- 解释指标：热图与医学关注区域一致性评分  

### 7.2 青少年坐姿智能检测模块（新增）

#### 功能目标

- 识别低头、弯腰、歪斜、久坐等不规范坐姿行为；  
- 给出实时提醒与阶段性风险统计；  
- 形成可追踪的行为改进报告。  

#### 技术路线（后续代码实现）

- 视频帧输入 -> 人体关键点估计 -> 姿态角度计算 -> 规则引擎评分 -> 风险等级与提示输出  
- 输出维度：单次告警、日均不良姿态时长、连续高风险时段统计  

#### 与骨骼影像模块的融合价值

- 影像筛查回答“当前生理风险”；  
- 坐姿检测回答“日常行为风险”；  
- 二者联合形成“风险评估 + 干预建议 + 复评追踪”闭环。

## 九、预期成果与考核指标

### 9.1 成果形式

- 原型系统：骨骼影像辅助分析 + 坐姿检测联动演示  
- 文档成果：技术方案、评测报告、部署说明、治理文档  
- 竞赛成果：挑战杯路演PPT、答辩讲稿、问答题库  

### 9.2 考核指标（建议）

- 算法性能：骨骼影像核心任务AUC达到0.85及以上，重点类别敏感度达到0.88及以上。  
- 行为检测：坐姿异常识别准确率达到0.90及以上，误报率控制在0.10以下。  
- 干预效果：连续干预后，不良姿态日均时长下降20%及以上。  
- 提前发现：联合评分相对单模块识别，高风险个体提前发现率提升10%及以上。  
- 工程落地：系统端到端可运行，关键流程可复现，主要接口稳定可用。  
- 治理合规：脱敏、权限、审计、问题沉淀机制执行到位，文档同步率达到100%。  

## 十、风险评估与应对策略

### 风险1：影像与行为模块跨度较大
- 应对：明确同属“青少年骨骼健康”问题域，采用模块化研发与分阶段验收。

### 风险2：坐姿检测实现复杂度高
- 应对：先规则引擎MVP，再逐步引入学习型优化策略，控制实现风险。

### 风险3：数据合规与隐私风险
- 应对：执行分级脱敏、最小权限、访问审计；校内数据必须审批后接入。

### 风险4：项目承诺过大影响可信度
- 应对：按“必达项（骨骼影像MVP）+挑战项（坐姿联动增强）”分层承诺。

## 十一、伦理合规与治理机制

- 严格执行匿名化处理，禁止存储直接身份信息。  
- 坚持最小权限原则，角色访问边界清晰。  
- 关键操作保留审计记录，满足追溯需求。  
- 执行项目治理硬约束：**每次新增云函数，先检查并更新数据库文档与云函数权限文档**。  
- 执行问题复盘机制：**每次问题修复后必须更新问题修改总结文档**。
