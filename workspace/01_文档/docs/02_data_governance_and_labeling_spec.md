# 数据治理与标注规范（SOP）

## 1. 明确结论

采用“**分级脱敏 + 标签标准化 + 患者级切分 + 质控闭环**”的数据治理方案，确保合规、可复现、可扩展。

## 2. 数据合规与脱敏规范

### 2.1 脱敏要求

- 移除或哈希化患者标识字段（姓名、证件号、住院号、手机号）。
- 清理DICOM头中可识别隐私字段（PatientName、PatientID等）。
- 文本报告执行PHI清洗（规则+人工抽检）。
- 所有样本使用随机UUID作为唯一索引。

### 2.2 访问控制

- 采用最小权限原则：训练集只读、标签修改需审批。
- 记录数据访问日志（时间、人员、用途）。
- 导出数据必须保留脱敏审计记录。

## 3. 标签体系（统一字典）

## 3.1 主任务标签

- `normal`: 正常
- `pneumonia`: 肺炎
- `nodule`: 肺结节
- `pleural_effusion`: 胸腔积液
- `pneumothorax`: 气胸

支持多标签并存，标签存储建议采用多热编码（multi-hot）。

### 3.2 不确定标签策略

- 公开数据中的 `uncertain` 先映射为 `ignore`。
- 训练时提供两条配置：
  - 配置A：`ignore`样本不参与该标签损失；
  - 配置B：按软标签（如0.3/0.7）参与训练。
- 评估时固定使用配置A，保证可比性。

## 4. 标注流程

1. 初标：医学背景成员按标签手册标注。
2. 复核：至少20%样本进行二次复核。
3. 仲裁：冲突样本由高年级医学顾问或合作医生裁决。
4. 冻结：每个里程碑生成一次“冻结标签版本”。

## 5. 数据切分与防泄露

- 按**患者级**切分训练/验证/测试（推荐7:1:2）。
- 同一患者跨时间检查必须进入同一数据子集。
- 外部验证集（如MIMIC外）单独保留，禁止参与调参。
- 固化随机种子，记录split清单文件以确保复现实验。

## 6. 数据质控指标

- 标签一致性：Cohen's Kappa >= 0.75（抽检集）
- 图像可用率：可读取样本比例 >= 99%
- 标签缺失率：主任务标签缺失 <= 1%
- 元数据完整率：年龄/性别等结构化字段完整 >= 95%

## 7. 风险点与后续动作

- 风险1：标注人力不足  
  后续动作：引入主动学习优先标注“高不确定样本”。

- 风险2：公开数据标签偏差  
  后续动作：构建小规模高质量“金标准验证集”。

- 风险3：多中心域偏移  
  后续动作：增加外部验证与分层评测报告。
