# 坐姿检测模块（占位）说明

## 1. 明确结论

坐姿检测已作为独立 **`pipeline: posture_stub`** 模块接入配置（`src/config/modules/posture_stub.yaml`），当前为**接口与配置桩**：不加载 MMCANet，不运行关键点与规则算法；返回固定占位结构，便于前端与联调预留。

## 2. 行为维度（占位标签）

与 `infer.class_names` 一致，共五维：

- 低头过度  
- 弯腰驼背  
- 躯干歪斜  
- 久坐超时提示  
- 视距过近提示  

## 3. HTTP 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/posture/info` | 模块元数据与维度列表（需 API Key，与主接口一致） |
| POST | `/posture/analyze` | 可选上传单帧图；当前返回全 0 占位分数 |

主流程影像接口 `/infer`、`/infer_with_image` 在 **`active_module: posture_stub`** 时亦返回占位说明，不生成 Grad-CAM。

## 4. 训练与评测

`train.py`、`evaluate.py` 在检测到 `pipeline != mmc_net` 时**直接退出**，避免误用 MMC 脚本训练坐姿占位。

## 5. 后续接入建议

1. 新建 `pipeline: posture_net` 或保留 stub 并增加子包实现推理；  
2. 在 `posture_stub.yaml` 同目录增量配置或替换模块 ID；  
3. 更新云数据库、权限文档及本记录。
